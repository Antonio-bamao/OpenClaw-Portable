# OpenClaw 便携 U 盘版 · 产品架构设计文档

## 0. 项目概述

<aside>
🎯

**产品定位：** 面向非技术用户的 OpenClaw 便携 U 盘版。插上即用、双击启动、WebUI 操作，零命令行。

**交付形态：** 实体 U 盘（64GB），内含完整运行时 + 启动器 + 预置配置 + 低风险技能包。

**目标平台：** Windows 10/11（64 位）

**商业模式：** 卖 U 盘实体 + 提供售后指引，用户自备模型 API Key。

</aside>

## 1. 架构总览

### 1.1 组件分层

| **层级** | **组件** | **说明** | **位置** |
| --- | --- | --- | --- |
| A. 启动器 | OpenClawLauncher.exe | GUI 控制面板，负责启动/停止/状态/引导/诊断 | U 盘根目录 |
| B. 运行时 | node.exe + OpenClaw 程序文件 | 内置 Node 22.x，锁定 OpenClaw 版本 | `runtime/` |
| C. 配置与数据 | .openclaw 状态目录 | 配置、会话、skills、渠道登录态 | `state/` |
| D. 缓存与日志 | 临时文件 + 日志 | 写入 `%TEMP%\OpenClawPortable\` | 系统临时目录 |
| E. WebUI | Control UI（gateway 内置） | 本地 Web 管理界面 | `http://127.0.0.1:<port>` |

## 2. 启动器设计（OpenClawLauncher.exe）

### 2.1 技术选型

<aside>
💡

**推荐方案：** 用 Python + PyInstaller 打包成 EXE（你已有这个经验，复用你的 MiniCamPro 打包流程）。

启动器本身只是一个"控制面板 GUI + 子进程管理"，不需要复杂框架。

GUI 框架推荐：**PyQt5/PySide6**（界面好看）或 **CustomTkinter**（轻量，打包体积小）。

</aside>

### 2.2 启动器功能清单

| **功能** | **优先级** | **说明** |
| --- | --- | --- |
| 🟢 启动 / 停止 / 重启 | P0 | 管理 gateway 子进程生命周期 |
| 🟢 打开 WebUI | P0 | 自动打开默认浏览器到正确端口 |
| 🟢 首次引导向导 | P0 | 设置密码 → 选 Provider → 填 Key → 测试连接 |
| 🟢 状态面板 | P0 | 运行状态 / 端口 / 运行时长 |
| 🟢 端口冲突处理 | P0 | 检测端口占用 → 自动换端口 → 通知用户 |
| 🟢 离线模式提示 | P0 | 检测无 Key 或无网络时弹提示，UI 可正常浏览但不能对话 |
| 🟡 渠道管理面板 | P1 | 微信/飞书/QQ/企微 的启用/配置/连接/断开 按钮 |
| 🟡 一键导出诊断包 | P1 | 打包日志 + 脱敏配置 + 版本信息 → zip |
| 🟡 检查更新 | P1 | 联网检查新版本 → 下载更新包 → 一键覆盖 |
| 🟡 恢复出厂设置 | P1 | 重置配置到初始状态（保留/不保留会话可选） |
| 🔵 托盘最小化 | P2 | 关闭窗口 → 缩到系统托盘 → 右键菜单操作 |
| 🔵 开机自启（可选） | P2 | 注册开机启动项（仅当 U 盘已插入时） |

## 3. 首次启动引导流程

### 3.1 流程图

```
双击 OpenClawLauncher.exe
        │
        ▼
  检测 state/openclaw.json 是否存在
        │
   ┌────┴────┐
   │ 不存在   │ 存在
   │（首次）  │
   ▼         ▼
 首次向导   正常启动
   │
   ▼
 Step 1: 设置 WebUI 管理密码
   │      （强制，不可跳过）
   ▼
 Step 2: 选择模型 Provider
   │      （下拉列表，默认选"通义千问"）
   │      选项：通义千问 / DeepSeek / OpenRouter / 自定义
   ▼
 Step 3: 填写 API Key
   │      （附"去哪申请 Key"的链接/二维码）
   │      （可跳过 → 进入离线模式）
   ▼
 Step 4: 测试连接
   │      （发一条测试消息验证 Key 有效性）
   │      成功 → "✅ 连接成功"
   │      失败 → "❌ Key 无效或网络问题，请检查"
   ▼
 Step 5: 渠道接入（可选，可跳过）
   │      "你想现在接入聊天平台吗？"
   │      微信 / 飞书 / QQ / 企微
   │      跳过 → 以后在 WebUI 里配
   ▼
 完成 → 自动启动 gateway → 打开 WebUI
```

### 3.2 引导界面要求

- 每一步都有"上一步/下一步"导航
- 每一步都有简短文字说明（不超过 3 句话）
- 敏感信息（密码/Key）输入框有 👁 切换明文/密文
- "去哪申请 Key"要给出具体 URL + 简要截图说明（放在 `assets/guide/` 里）
- 引导完成后生成 `state/openclaw.json` + `state/.env`
- 引导可从启动器主界面重新打开（"重新配置"按钮）

## 4. 安全策略（默认值）

<aside>
🔒

**原则：默认最保守，用户手动放宽。** 你卖的是工具不是后门，安全做不好法律风险极大。

</aside>

### 4.1 默认安全配置

| **配置项** | **默认值** | **理由** |
| --- | --- | --- |
| gateway.bind | `loopback`（仅本机） | 不暴露公网，防止被远程访问 |
| gateway.auth.mode | `token`（首次向导设置） | WebUI 必须认证才能使用 |
| channels.*.dmPolicy | `pairing` | 只有配对过的设备才能私聊 |
| channels.*.groupPolicy | `allowlist` | 默认不响应群聊，需手动添加白名单 |
| skills 权限 | 仅启用低风险技能 | 不默认启用文件删除/命令执行/浏览器操作 |
| 日志脱敏 | 日志中不打印 API Key / Token 全文 | 防止诊断包泄露用户密钥 |

### 4.2 日志中的脱敏规则

- API Key：只显示前 4 位 + `***` + 后 4 位
- Token：只显示前 8 位 + `***`
- 用户消息内容：诊断包中默认不包含（可选导出）

## 5. 模型 Provider 模板库

### 5.1 预置 Provider（国产优先）

| **Provider 名称** | **baseUrl** | **协议** | **申请 Key 地址** |
| --- | --- | --- | --- |
| 通义千问（推荐默认） | `https://dashscope.aliyuncs.com/compatible-mode/v1` | OpenAI 兼容 | [bailian.console.aliyun.com](http://bailian.console.aliyun.com) |
| DeepSeek 官方 | `https://api.deepseek.com/v1` | OpenAI 兼容 | [platform.deepseek.com](http://platform.deepseek.com) |
| OpenRouter（海外模型聚合） | `https://openrouter.ai/api/v1` | OpenAI 兼容 | [openrouter.ai](http://openrouter.ai) |
| 自定义（高级） | 用户填写 | OpenAI 兼容 | — |

### 5.2 Provider 模板机制

- 预置模板存放在 `state/provider-templates/`
- 每个模板是一个 JSON 文件，包含：baseUrl、默认模型列表、api 协议类型
- 用户选择 Provider 后：自动填充 baseUrl 和模型列表，用户只需填 Key
- 模板随更新包迭代（新版本可增加 Provider，不影响用户已有配置）

## 6. 渠道接入设计（微信 / 飞书 / QQ / 企微）

### 6.1 总体策略

- 渠道插件预装在 `runtime/plugins/` 目录
- 启动器提供"渠道管理"面板（或直接在 WebUI 里操作）
- 每个渠道有独立的"启用/配置/连接/断开/删除"按钮
- 每个渠道的接入引导做成独立的 HTML 帮助页（放 `assets/guide/`）

### 6.2 各渠道接入流程

| **渠道** | **接入方式** | **用户操作** | **风险提示** |
| --- | --- | --- | --- |
| 微信 | 官方 ClawBot 插件（扫码） | 点"连接微信" → 弹二维码 → 扫码 | 需微信 8.0.70+；换电脑需重新扫码 |
| 飞书 | 飞书开放平台 Bot（WebSocket） | 填 App ID + App Secret → 测试 | 需要在飞书开放平台创建应用 |
| QQ | QQ 官方机器人 / 社区方案 | 填 Bot AppID + Token | QQ 官方机器人仅限频道/群；个人号接入有封号风险 |
| 企业微信 | 企微自建应用 Bot | 填 CorpID + AgentID + Secret | 需企业管理员权限创建应用 |

### 6.3 渠道状态管理

- 每个渠道的登录态/token 保存在 `state/channels/<渠道名>/`
- 换电脑时提示"检测到新设备，以下渠道需要重新登录"并列出受影响渠道
- 微信登录态尤其脆弱（绑定设备），需特别提示

## 7. 内置技能包（低风险）

### 7.1 预装技能清单

| **技能名称** | **功能** | **风险等级** |
| --- | --- | --- |
| 文档总结 | 总结 / 提炼 / 翻译文本内容 | 🟢 低 |
| 表格生成 | 根据描述生成 CSV / Markdown 表格 | 🟢 低 |
| 日程管理 | 提醒、待办、日历事件 | 🟢 低 |
| 网页摘要 | 给一个 URL 返回摘要 | 🟢 低（只读） |
| 知识问答 | 基于用户上传的文档做 RAG 问答 | 🟡 中低（涉及文件读取，但只读） |

### 7.2 高危技能默认关闭

以下技能**不**默认启用，用户需在 WebUI 中手动开启并确认风险提示：

- 文件写入 / 删除
- 任意 Shell 命令执行
- 浏览器自动化（模拟点击/下单）
- 代码执行沙箱

## 8. 缓存与日志策略

### 8.1 写入路径

| **数据类型** | **写入位置** | **理由** |
| --- | --- | --- |
| 配置 / 会话 / Skills / 渠道态 | U 盘 `state/` | 必须跟着 U 盘走（便携核心） |
| 运行日志 | `%TEMP%\OpenClawPortable\logs\` | 减少 U 盘写入，提升速度 |
| 模型缓存 / 临时文件 | `%TEMP%\OpenClawPortable\cache\` | 大量临时数据不适合写 U 盘 |

### 8.2 日志管理

- 按日期分割：`openclaw-2026-04-05.log`
- 单文件上限 50MB，超出自动轮转
- 保留最近 7 天日志，自动清理旧日志
- 启动器底部小字显示：`缓存路径: C:\Users\xxx\AppData\Local\Temp\OpenClawPortable\`

## 9. 版本管理与更新策略

### 9.1 版本号规则

- 格式：`v<年>.<月>.<修订号>`，例如 `v2026.04.1`
- 每月跟进 OpenClaw 上游更新，测试稳定后发布

### 9.2 更新流程

```
用户点"检查更新"
       │
       ▼
联网查询最新版本号（你需要一个简单的版本检查 API / GitHub Release）
       │
  ┌────┴────┐
  │ 无更新   │ 有更新
  │          │
  ▼          ▼
 "已是最新"  显示更新日志 → 用户确认
                │
                ▼
          下载更新包（zip）
                │
                ▼
          备份当前 runtime/ 到 runtime.bak/
                │
                ▼
          解压新 runtime/ 覆盖
                │
                ▼
          重启 gateway
                │
                ▼
          验证启动成功 → 删除 bak
          启动失败 → 自动回滚 bak → 提示用户
```

### 9.3 回滚机制

- 更新前自动备份 `runtime/` → `runtime.bak/`
- `state/` 永远不被更新覆盖（用户数据安全）
- 启动器检测"更新后首次启动失败" → 自动回滚 → 弹窗提示

## 10. 售后与诊断体系

### 10.1 一键诊断包内容

| **内容** | **是否脱敏** |
| --- | --- |
| version.json（软件版本） | 否 |
| openclaw.json（主配置） | ✅ 脱敏：Key/Token 只保留前 4 位 |
| 最近 3 天运行日志 | ✅ 脱敏：用户消息内容不包含 |
| 端口占用检测结果 | 否 |
| 系统信息（OS 版本/内存/磁盘） | 否 |
| 渠道连接状态 | ✅ 脱敏：token 只保留前 8 位 |

### 10.2 常见问题自动检测

启动器在启动时自动检测并提示：

- 端口被占 → "端口 18789 被占用，已自动切换到 18790"
- 无网络 → "当前无网络连接，已进入离线模式"
- Key 过期/无效 → "API Key 验证失败，请在设置中更新"
- U 盘空间不足 → "U 盘剩余空间不足 500MB，请清理"
- 杀毒软件拦截 → "如果程序无法启动，请将 OpenClawLauncher.exe 添加到杀毒软件白名单"（常驻提示）

## 12. 关于代码签名的说明

<aside>
⚠️

**你说你之前自己打包的 EXE 能打开——没错，但那是你自己电脑。**

当你把 EXE 发给别人（尤其是通过 U 盘/网盘分发）时，Windows SmartScreen 会弹出"Windows 已保护你的电脑"蓝色警告框，很多小白看到这个直接不敢点了，甚至以为是病毒。

**你有三条路：**

1. **不签名 + 写清楚"如何放行"教程**（成本最低，但售后会多）
2. **买代码签名证书（约 ¥500-2000/年）**（一劳永逸，用户无感）
3. **用 Inno Setup 做安装包**（比裸 EXE 好一点，但 SmartScreen 仍会拦未签名的安装包）

**建议 MVP 阶段先走路线 1**：在 README.txt 和首次引导里放截图教用户怎么点"仍要运行"。卖出量大了再考虑签名。

</aside>
